## 레이아웃 재설계

### 변경 전 → 후
- **Before**: flex row `[MapPanel | CardPanel]` — 패널 크기에 따라 지도가 밀림
- **After**: 지도 `absolute inset-0` 전체배경 + 좌/우 패널 오버레이

### 최종 z-index 계층
| 레이어 | z-index | 요소 |
|--------|---------|------|
| Layer 0 | `z-0` | 지도 (MapPanel wrapper, stacking context 격리) |
| Layer 1 | `z-10` | 좌측 패널 (가중치), 우측 패널 (추천 카드) |
| Layer 2 | `z-20` | 패널 토글 버튼 (main 직계 자식) |
| Layer 3 | `z-[60]` | InfoTooltip 말풍선 |

### Steelman — 적용한 처방의 반대 주장

- **#6 Portal화는 과한가?** wrapper의 `transform` 제거가 더 가벼우나 패널 슬라이드 애니메이션이 끊김. Portal이 안전한 선택.
- **#4 칩 클릭 시 입력 비우기는 공격적인가?** 사용자가 단순 비교 의도로 칩만 눌러도 입력이 날아감. 대안: 입력 유지 + hint를 회색 처리. 다음 이터레이션 검토 대상.
- **#1 resize 리스너는 필요한가?** 오버레이 구조라 panel collapse는 영향 없지만, 헤더 표시/숨김·모바일 회전·devtools 토글에서 가치 있음 → 유지.
