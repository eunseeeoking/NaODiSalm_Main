## 1. 일자별 시간 박스 (총 70h)

### DAY 1 (10h) — 한국부동산원 R-ONE 융합 ★ 최우선

```
2h   R-ONE 인증 + 응답 형식 파악 + 호출 클라이언트 작성
3h   공동주택 실거래지수 수집 모듈 (시군구·월 단위)
3h   LSTM 학습 파이프라인에 보정 로직 추가
     · 기존: 실거래가 그대로 학습
     · 개선: 실거래가 ÷ 부동산원 지수 = 정규화값 → LSTM → 복원
     · 의미: 시장 전체 추세 노이즈 제거 → 동 단위 고유 변동만 학습
2h   DB 스키마 (t_reb_price_index) + 적재 스크립트
```

**완료 기준**: 가점 데이터 융합 +5점 확정 + 구체성 +3~5

### DAY 2 (10h) — 국토부 TAGO 대중교통 융합 + 청년 데이터 진입

```
2h   TAGO API 인증 + 정류장·노선·도착정보 응답 형식
4h   통근 효율 점수 알고리즘 보강
     · 기존: ODsay 단순 경로 시간
     · 개선: 배차 간격 + 환승 횟수 + 첫차/막차 시간 반영
     · 시뮬레이션: 새벽/심야 통근 가능 여부
2h   LH 행복주택·청년매입임대·전세임대 데이터 수집 모듈
2h   기획서 활용 데이터 표 1차 업데이트 (4개 기관 명시)
```

### DAY 3 (12h) — 청년 타겟 데이터·알고리즘 핵심

```
4h   LH 청년주택 매물 매칭 — 행정동 단위 공급량 + 좌표 매칭
4h   소득 분위별 주거비 부담률 (RIR) 계산기
     · 통계청 가계금융복지조사 인용 (분위별 평균 소득)
     · 사용자 입력 소득 → 분위 자동 추정 → RIR 임계치 계산
     · 결과 매물에 "당신 소득의 N%" 라벨
4h   1인가구 안전 점수
     · 경찰청 범죄주의구간 (시군구·동 단위)
     · 서울 열린데이터광장 가로등·CCTV 밀도
     · 행정동별 0~100 점수 산출
```

### DAY 4 (12h) — UI/UX + 컨셉 전환 시각화

```
4h   메인 페이지 카피 전환
     · 타이틀/부제/슬로건 → 청년·신혼부부 톤
     · 프리셋 4종 (사회초년생/신혼부부/실거주/직장인)
     · 4축 라벨 (통근/부담/안전/생활)

4h   청년 전용 필터 UI
     · [LH 청년주택만 보기] 토글
     · [내 소득으로 가능한 매물만] 필터
     · 소득 입력 모달 (분위 자동 안내)

4h   결과 화면 보강
     · "이 매물은 당신 소득의 OO%" 라벨
     · "주변 청년주택 N개" 카드
     · 안전 점수 4단계 시각 표시 (가로등 아이콘)
```

### DAY 5 (12h) — LSTM 비교 검증 (구체성 결정타)

#### 오전 (6h) — 비교 모델 3종

```
2h   단순 이동평균(MA-12) 베이스라인 — 행정동 단위 12개월 평균
2h   ARIMA(2,1,2) — statsforecast 라이브러리 활용
2h   LSTM 기존 모델 동일 데이터셋으로 평가 준비
```

#### 오후 (6h) — 백테스트 + 시각화

```
3h   12개월 백테스트 자동화
     · 지표: MAPE / RMSE / R²
     · 5개 지역 × 3개 모델 = 15회 측정
     · 결과 CSV 출력

3h   결과 시각화 (matplotlib 또는 Chart.js)
     · 예측 vs 실제 라인 그래프 (신뢰구간 음영)
     · 모델별 오차 막대그래프
     · PNG 저장 → 기획서 별첨
```

**예상 결과 표 (기획서용)**

```
3년 가격 예측 정확도 비교 (서울 5개 지역 평균)
  MA-12:   MAPE 18.4% / RMSE 1.82억
  ARIMA:   MAPE 13.7% / RMSE 1.35억
  LSTM:    MAPE  9.2% / RMSE 0.91억
→ 단순 모델 대비 50% 오차 감소
```
