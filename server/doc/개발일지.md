# 🚀 프로젝트 비하인드 스토리: "내가 답답해서 직접 만든 주거선택 플랫폼"

> **본 문서는 개발 과정에서 발생한 기획적 고뇌, 기술적 돌파구, 비즈니스 모델(BM) 수립 및 사회적 가치 검증 과정을 기록한 개발 일지입니다.**

## 🪵 1. 시작점: "판단은 사용자 몫"이라는 책임 전가에 대한 의문

### 💡 문제 의식 (Pain Point)
기존의 프롭테크 플랫폼들은 가공되지 않은 raw 데이터나 수많은 부동산 그래프를 한 화면에 쾅 던져놓고 **"데이터는 투명하게 줬으니, 판단은 알아서 하세요"**라는 식으로 유저에게 책임을 전가하는 경향이 있음.
이로 인해 일반 사용자는 **정보 과부하(Information Overload)**에 걸려 오히려 의사결정에 장애를 겪음.

### 🎯 철학적 돌파구: '원시 데이터'가 아닌 '정제된 신호(Signal)'의 제공
유저에게 주도적인 판단력을 부여하면서도 과한 정보 주입을 막기 위해 **UI/UX 관점의 정보 계층화(Progressive Disclosure)**를 도입함.
1. **1단계 (신호):** AI가 계산한 추천 점수를 직관적인 '색상'과 '한 줄 요약'으로 먼저 제시하여 뇌 용량 소모를 최소화함.
2. **2단계 (근거):** 왜 그런 점수가 나왔는지 핵심 지표(통근, 자산방어력 등)를 게이지 바/그래프로 결합함.
3. **3단계 (투명성):** 더 깊은 데이터가 필요한 헤비 유저만 버튼을 눌러 LSTM Loss 값 백테스트 결과, 실거래가 추이 등 딥한 데이터를 확인하도록 설계함.

## 🛠️ 2. 기술적 워프(Warp) 및 아키텍처 구현

기획서의 타임라인을 앞서가며 주말과 새벽 세션을 통해 핵심 기술 스택을 빠르게 빌딩함.

### 📊 데이터 전처리: '수도권 중심 가중치' 제거
* **기존의 한계:** 일반적인 시계열 학습 시, 서울/수도권의 폭발적인 거래량과 상승률 데이터가 전체 모델을 오염(Bias)시켜 외곽 지역의 예측 정확도를 대폭 떨어뜨림.
* **해결 책:** 통계적으로 수도권 특수성 가중치를 발라내고 정규화(Normalization)한 깨끗한 데이터셋을 구축함. 이를 [지역 × 평형 × 연식]의 16개 세분화 축과 매칭함.

### 🧠 ML 및 인프라 분리 (3분할 모노레포)
* Render(백엔드 프리티어)의 자원 한계를 인지하고, 클라이언트(`Vercel`), API 서버(`Render`), ML 학습 모듈을 완벽하게 분리함. 무거운 TensorFlow.js LSTM 모델 학습은 로컬 자원을 활용하고 결과 데이터를 분산 DB(`TiDB Cloud`)에 찔러 넣는 아키텍처를 채택하여 인프라 비용을 제로(0)로 방어함.

### 🗺️ 공간정보 시각화: 등시선 폴리곤(Polygon) 및 뎁스 확장 UI
* 텍스트 위주의 단순 정보 제공을 탈피, 사용자 직장 위치 중심의 출퇴근 가능 범위를 지도 위에 색상이 들어간 **폴리곤(Isochrone) 영역**으로 실시간 시각화함. 
* 유저가 특정 지역을 클릭할 때 인터랙티브하게 뎁스가 확장되며 역대 실거래가 카드가 나타나는 프로덕션 레벨의 프론트엔드 UX를 구현함.

## 📱 3. 치명적 결함 발견 및 예외 처리 (QA 전략)

### ⚠️ 리스크 인지: 모바일 환경에서의 UX 파편화
공모전 심사위원들이 이동 중 모바일이나 태블릿으로 데모 링크에 접근할 시, 복잡한 지도 인터랙션과 확장 카드가 화면을 가리거나 깨질 수 있는 리스크를 발견함.

### 🛡️ 방어 및 보완 전략
1. **가성비 최적화:** 모바일 화면폭(768px 이하) 진입 시 지도를 `100vw/100vh`로 고정하고, 상세 카드는 기존 앱들이 채택하는 **하단 시트(Bottom Sheet) UX**로 CSS 덮어쓰기를 수행함. 복잡한 데이터는 모바일에서 과감히 숨김(`display: none`) 처리함.
2. **논리적 방어전략 구축:** 본 서비스는 고관여 자산 분석 도구이므로 데스크톱(PC) 환경이 메인 프로토타입임을 명시하되, 향후 PWA(Progressive Web App) 또는 React Native로 즉시 패키징할 수 있는 구조적 확장성을 기획서에 포함하여 기술적 신뢰도를 확보함.

## 💰 4. 비즈니스 모델(BM): "낮은 재방문율"에 대한 역발상

### ❓ 의문점
지역 추천이라는 서비스 특성상, 유저가 목적(지역 확정)을 달성하면 재방문(Retention)할 필요가 없음. 비즈니스 구조가 무너지는가?

### 🎯 역발상 BM (Conversion-based Model)
기존 부동산 앱처럼 트래픽을 가두어 광고를 보여주는 모델이 아닌, **'인생의 전환기(Trigger)에 딱 한 번 들어와 확신을 얻고 나가는 고효율 유틸리티'**로 정의함. 트래픽 유지 마케팅 비용과 서버 비용을 극도로 아끼는 Lean 구조를 지향하며 아래 3단계 BM을 구축함.

1. **B2B 중개 수수료 쉐어 (Affiliate):** 앱 내에서 최적의 지역(동/블록)을 확정한 초고품질 유저를 기존 매물 플랫폼(직방, 네이버 부동산 등)으로 조건이 자동 입력된 채 아웃링크 연동하여 전환 계약 시 리퍼럴(Referral) 수익을 쉐어함.
2. **프리미엄 레포트 (1회성 유료 결제):** 내 예산과 가중치가 정밀하게 분석된 20페이지 분량의 '나만의 주거 최적화 PDF 보고서'를 다운로드할 때 1회성 결제 유도.
3. **전후방 산업 큐레이션 광고:** '지역 확정'은 곧 '이사가 임박했다'는 강력한 유저 시그널임. 대출 상품(전세대출), 이사업체, 입주청소, 인테리어 등 타겟팅 효율이 극도로 높은 서드파티 광고 큐레이션을 매칭하여 유저당 평균 매출(ARPU)을 극대화함.

## 🕊️ 5. 최종 결론: 이 프로젝트가 지닌 사회적 가치

> **"네비게이션은 도로나 좁은 골목길을 넓혀주지 못하지만, 막히지 않는 가장 최적의 경로를 찾아내 운전자의 불안을 해소하고 시간을 구원합니다."**

비싸고 매물이 한정된 수도권 부동산 시장의 가혹한 현실을 이 앱이 직접 바꿀 수는 없음. 그러나 본 플랫폼은 아래와 같은 명확한 사회적 가치를 창출함.

1. **청년 가구의 정보 비대칭성 해소:** 생전 가본 적 없는 동네에서 수억 원의 보증금을 걸어야 하는 청년들에게 데이터 기반의 안전지대를 과학적으로 증명하여 '실패하지 않을 권리'를 제공함.
2. **국가 교통 정책(K-MaaS)의 민간 대중화:** 정부가 구축한 거대 복잡한 MaaS 인프라 개념을 개인의 '주거 선택'이라는 가장 피부에 와닿는 일상 영역으로 끌어내려 시각화함.
3. **삶의 시간 구원:** 길바닥에 버려지는 출퇴근 통근 시간을 비용과 가치로 환산해 줌으로써, 유저가 한정된 자원 속에서 **'가장 덜 후회할 수 있는 최적의 타협점'**을 찾고 인간다운 삶의 질을 확보할 수 있도록 돕는 진정한 나침반의 역할을 수행함.
